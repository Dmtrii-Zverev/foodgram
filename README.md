# Foodgram — backend сервиса рецептов

Backend часть сервиса для публикации и хранения рецептов.  
Пользователи могут публиковать рецепты, подписываться на авторов, добавлять рецепты в избранное и формировать список покупок.

Проект реализован как REST API сервис.

---

## Стек технологий

- Python
- Django
- Django REST Framework
- PostgreSQL
- Docker
- Nginx
- Gunicorn
- GitHub Actions (CI/CD)

---

## Основные возможности

- регистрация и аутентификация пользователей
- публикация рецептов
- подписки на авторов
- добавление рецептов в избранное
- формирование списка покупок
- фильтрация и пагинация рецептов
## Архитектура сервиса

Клиент взаимодействует с API через HTTP запросы.
Клиент
   |
REST API (Django + DRF)
   |
PostgreSQL

## Примеры API

### Создание рецепта

POST /api/recipes/

Пример запроса:

```json
{
  "name": "Паста",
  "ingredients": [
    {"id": 1, "amount": 200}
  ],
  "cooking_time": 20
}
 ```markdown
## Запуск проекта

Клонировать репозиторий:

git clone https://github.com/Dmtrii-Zverev/foodgram.git

Перейти в папку проекта:

cd foodgram

Запустить через Docker:

docker compose up
